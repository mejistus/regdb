项目与方法说明

这个项目做什么

本项目复现的是无监督可见光-红外行人重识别。输入是一组可见光图像和一组红外图像，训练时不使用人工身份标签，目标是学习同一个人在两种模态下都接近的特征表示。

评测时用一个模态作为查询，另一个模态作为图库，按特征距离检索同一身份。当前 RegDB 脚本记录的是 visible-to-thermal 方向。

为什么有多个 trial

RegDB 官方提供 10 组划分。每个 trial 都是一套不同的训练/测试拆分，论文通常报告 10 个 trial 的平均结果，以降低单次划分带来的偶然性。

本次 AMP fp16 复现按当前目标固定跑 trial 1-3，对比同环境下的 baseline、MDUE 和 MDUE+CGCF；最终表格会明确标出已完成 trial 数。

为什么有 stage1 和 stage2

stage1 是初始无监督聚类训练：分别对 RGB/IR 特征做聚类，把聚类编号当作伪标签训练模型，得到可用的初始 checkpoint。

stage2 从 stage1 checkpoint 继续训练，加入跨模态关联和二分图匹配，把 RGB 与 IR 中可能属于同一人的聚类对应起来，重点优化跨模态检索性能。

论文第 5 章创新点

MDUE 在特征抽取时保持 Dropout 随机性，多次前向估计样本表征方差，并把低方差样本赋予更高置信度。本次 AMP fp16 复现按论文设置 S=3、p=0.10。

CGCF 在 All-memory 原型构造时，不再简单平均 RGB/IR 中心，而是用模态置信度和跨模态一致性加权融合簇中心。

本次复现设置

环境使用 Python 3.12、PyTorch 2.6.0、单张 Tesla V100 16GB 和 AMP fp16。正式 AMP 组保持论文训练长度与聚类参数；stage1 使用 batch 64，stage2 使用 batch 32 以适配显存，表格中的 Batch 列会记录每个 run 的实际 batch size。

当前报告的主结论只比较同一 AMP fp16 环境下的 PCLHD baseline、PCLHD+MDUE 和 PCLHD+MDUE+CGCF，默认范围是 RegDB trial 1-3。

算法流程

本项目把无监督跨模态 ReID 拆成“单模态伪标签初始化”和“跨模态关联优化”两段。论文第 5 章的最终方法在此基础上加入 MDUE 样本置信度和 CGCF 跨模态中心融合。

  1. 1. 数据准备读取 RegDB/SYSU 的 RGB 与 IR 图像，按官方划分生成训练集、查询集和图库。
  2. 2. 特征初始化使用 ImageNet 预训练 backbone 提取两种模态的初始特征，并用 GeM/BNNeck 形成检索向量。
  3. 3. Stage1 聚类训练分别对 RGB 与 IR 特征做 DBSCAN 聚类，把聚类 ID 当作伪标签，用 DCL 和 ClusterMemory 训练初始模型。
  4. 4. MDUE 特征抽取开启 MC-Dropout，原图和水平翻转图各做多次随机前向，取均值作为稳健特征，并由方差得到样本置信度。
  5. 5. CGCF 原型融合对同一 All-cluster 的 RGB/IR 中心，用模态置信度和中心相似度计算权重，得到更可靠的统一跨模态原型。
  6. 6. Stage2 跨模态关联加载 stage1 checkpoint，构造 RGB、IR、All 三组 memory，通过跨模态相似度和二分图匹配建立 RGB-IR 原型对应关系。
  7. 7. 评估与汇总用最终 stage2 checkpoint 做 visible-to-thermal 与 thermal-to-visible 检索，汇总 Rank-1、mAP、mINP 和逐 trial 均值。

实现审计与参数对齐

这部分记录当前复现代码和论文方法的对应关系，便于解释“代码里是否真的包含 MDUE/CGCF”。

原始代码状态
初始提交 d9145c9 中已有 extract_features(..., mc_drop) 和 confidence_fusion_features() 这类辅助函数雏形，但 train_regdb.py 没有暴露 --mdue-samples、--use-cgcf，也没有把它们接入 RegDB stage1/stage2 训练路径。
本次接入
当前版本在 train_regdb.py 中加入 MDUE/CGCF 开关：stage1 与 stage2 的聚类特征抽取使用 S=3 的 MC-Dropout 均值特征；stage2 的 All-memory 原型在开启 CGCF 时调用置信度引导的跨模态中心融合。
Dropout 修正
clustercontrast/models/agw.py 使用 mc_dropout_active 控制 Dropout，仅在不确定性采样阶段激活；普通评估和 baseline 不受随机 Dropout 干扰。
论文参数
full AMP 组保持 epochs=50、iters=100、eps=0.3、momentum=0.1、num_instances=16；baseline 使用 S=1、dropout=0、不开 CGCF，MDUE/CGCF 组使用 S=3、dropout=0.10。
环境差异
允许差异集中在 Python 3.12、PyTorch 2.6.0、V100 16GB 和 AMP fp16。stage2 使用较小 batch 以适配显存，实际值在表格中记录。
公式注意
当前 CGCF 按论文公式同时用跨模态一致性 s_vr 乘到 RGB/IR 两侧权重；两侧共同归一化时，差异性权重主要来自 c_rgb 与 c_ir，这会影响后续解释 CGCF 增益的幅度。

术语解释

run
一次具体训练任务。本次主实验包含 3 个方法配置和 3 个 RegDB trial；每个 trial 有 stage1 和 stage2，所以期望看到 18 个 stage-level run。
epoch
完整遍历一次当前训练采样流程后的评估点。本报告逐 epoch 记录 Rank 和 mAP，便于观察训练走势。
checkpoint
训练过程中保存的最佳模型权重。本报告的 Ckpt 字段对应日志里记录的 best epoch。
Rank-1 / R1
查询图像检索结果的第一名就是正确身份的比例。越高越好，是行人重识别最直观的指标。
Rank-5/10/20
正确身份出现在前 5、10、20 个检索结果中的比例。它们衡量较宽松的召回能力。
mAP
mean Average Precision，综合衡量排序列表中所有正确匹配的位置，越高说明整体排序越好。
mINP
mean Inverse Negative Penalty，更关注最后一个正确匹配在排序中的位置，反映困难样本检索质量。
Assoc
stage2 中 RGB/IR 聚类成功建立跨模态关联的比例。它用于观察跨模态匹配是否逐步稳定。
MDUE
MC-Dropout Uncertainty Estimation。通过多次随机前向计算样本特征均值与方差，把不稳定样本转成低置信度。
CGCF
Confidence-Guided Cross-modal Center Fusion。用样本置信度和 RGB/IR 中心一致性加权融合跨模态原型。
AMP
Automatic Mixed Precision。训练前向和反向使用 fp16 混合精度，以降低显存占用并提升吞吐。
Best R1
该 run 中 Rank-1 最高的 epoch，对应主要 checkpoint 选择依据。
Best mAP
该 run 中 mAP 的最高值。本报告把 Best R1 和 Best mAP 分开计算，因为最高 Rank-1 和最高 mAP 不一定发生在同一个 epoch。
